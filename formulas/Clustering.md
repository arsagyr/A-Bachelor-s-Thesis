# Описание методов кластеризации из файла `clustering_calc.py`

## 1. `decimal_to_float(value)` – преобразование Decimal в float

**Описание:**  
Вспомогательный статический метод, который проверяет наличие метода `to_eng_string` (характерного для типа `Decimal` из модуля `decimal`). Если метод существует, значение преобразуется в `float`, иначе возвращается без изменений. Необходим для корректной работы `numpy` и `scikit-learn`, которые не поддерживают `Decimal`. Используется внутри `prepare_features`.

**Формула:**  
Не применима (преобразование типа).

## 2. `prepare_features(df)` – подготовка признаков для кластеризации

**Описание:**  
Принимает `DataFrame` с колонками `country_id`, `country_name`, `export_value`, `import_value`, `gdp_value`. Преобразует `Decimal` в `float`. Вычисляет производные показатели: отношение экспорта к ВВП (`export_per_gdp`), импорта к ВВП (`import_per_gdp`). Для учёта масштаба добавляет логарифмированные признаки через `log1p`: `log_export`, `log_import`, `log_gdp`. Итоговый набор признаков включает 5 колонок: логарифм ВВП, логарифм экспорта, логарифм импорта, доля экспорта в ВВП (%), доля импорта в ВВП (%). Возвращает матрицу признаков и список названий. Пропуски и бесконечности заменяются нулями.

**Формулы:**
\[
\text{export\_per\_gdp} = \frac{\text{export\_value}}{\text{gdp\_value}} \times 100
\]
\[
\text{import\_per\_gdp} = \frac{\text{import\_value}}{\text{gdp\_value}} \times 100
\]
\[
\text{log\_x} = \ln(1 + x), \quad x \in \{\text{export\_value}, \text{import\_value}, \text{gdp\_value}\}
\]

## 3. `find_optimal_clusters(features, max_k=6)` – определение оптимального числа кластеров

**Описание:**  
Для заданной матрицы признаков вычисляет инерцию (сумму квадратов расстояний до центров кластеров) для \(k\) от 2 до `max_k` (но не более \(n_{\text{samples}}-1\)). Инерция используется для метода «локтя»: определяется \(k\), при котором относительное изменение инерции становится меньше 10%. Также вычисляется силуэтный коэффициент (только для информации). Если выборка слишком мала (\(n_{\text{samples}} < 3\)), возвращает `optimal_k = 1`. По умолчанию при невозможности найти «локоть» устанавливается `optimal_k = 3`. Результат содержит оптимальное \(k\), списки инерций, силуэтных оценок и соответствующих \(k\).

**Формулы:**
\[
\text{inertia}(k) = \sum_{i=1}^{n} \sum_{j=1}^{k} \mathbf{1}_{c(i)=j} \cdot \|\mathbf{x}_i - \boldsymbol{\mu}_j\|^2
\]
\[
\Delta_k = \frac{\text{inertia}(k-1) - \text{inertia}(k)}{\text{inertia}(k-1)} \times 100\%
\]
Оптимальное \(k\) – наименьшее, для которого \(\Delta_k < 10\%\).

## 4. `perform_clustering(features, n_clusters=3)` – выполнение K-Means кластеризации

**Описание:**  
Принимает матрицу признаков и желаемое число кластеров. Проверяет, достаточно ли данных (не менее `n_clusters`). Стандартизирует признаки с помощью `StandardScaler` (приводит к среднему 0 и дисперсии 1). Применяет K-Means с `n_init=10` и фиксированным `random_state=42`. После получения меток кластеров вычисляет средний `log_gdp` (первый признак) для каждого исходного кластера, сортирует кластеры по убыванию этого показателя и переименовывает их в «Передовые», «Средние», «Отстающие» (для трёх кластеров). Метки перекодируются в соответствии с новым порядком (0 – самый богатый кластер). Возвращает словарь с флагом успеха, количеством кластеров, новыми метками, информацией о кластерах (`cluster_id`, `type`, `size`), а также объекты `scaler` и `model` для использования при предсказании на новых данных.

**Формулы:**
\[
\text{Стандартизация:} \quad z_i = \frac{x_i - \mu}{\sigma}
\]
\[
\text{K-Means минимизирует:} \quad \min_{\boldsymbol{\mu}_j} \sum_{i=1}^{n} \min_j \|\mathbf{z}_i - \boldsymbol{\mu}_j\|^2
\]
\[
\text{Ранжирование кластеров:} \quad \text{rank}(j) \sim \text{сортировка} \left( \frac{1}{|C_j|} \sum_{i \in C_j} \text{log\_gdp}_i \right)
\]
